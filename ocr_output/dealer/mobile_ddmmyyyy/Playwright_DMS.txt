================================================================================
Playwright_DMS — Hero Connect / Siebel DMS automation (reference)
================================================================================
Source: backend/app/services/siebel_dms_playwright.py
Entry: run_hero_siebel_dms_flow()

On disk (this copy): ocr_output/dealer/mobile_ddmmyyyy/Playwright_DMS.txt
Runtime automation traces (e.g. DMS_Form_Values.txt) use the same shape with a
numeric dealer id and dated subfolder: ocr_output/<dealer_id>/<mobile_ddmmyy>/

Stage 5 in code is the nested helper stage_5_vehicle_flow() (vehicle search,
scrape, In-Transit receipt, Pre Check, PDI).

This file describes what Playwright does during real DMS fill: navigation targets
(env-backed URLs), form fields and which dms_values keys supply them, toolbar
clicks, and branch decisions (customer found / not found, In Transit, etc.).

All navigation uses page.goto(..., wait_until="domcontentloaded") to absolute
Siebel URLs. Siebel often nests content in iframes; selectors run with optional
DMS_SIEBEL_CONTENT_FRAME_SELECTOR / auto iframe discovery.

--------------------------------------------------------------------------------
A. INPUT DATA (dms_values dict keys used by this flow)
--------------------------------------------------------------------------------
  mobile_phone       → Contact Find + enquiry-form mobile; grid match needle
  first_name         → Basic enquiry (First Name inputs)
  last_name          → Basic enquiry (Last Name inputs)
  address_line_1     → Basic enquiry (Address / Street; truncated ~120 chars)
  state              → Basic enquiry (State <select>)
  pin_code           → Basic enquiry (Postal / ZIP / PIN inputs)
  landline           → skip_find path only: Work / Alternate / Landline inputs
  father_husband_name→ Stage 4 care-of (Father / Husband / Parent inputs; max 255)
  relation_prefix    → Stage 4 care-of (Relation / S/O <select>)
  key_partial        → Vehicle list: Key # field (not Keyboard)
  frame_partial      → Vehicle list: Chassis / Frame / VIN field
  engine_partial     → Vehicle list: Engine field
  dms_contact_path   → Controls Stage 1 match override (see decisions below)

--------------------------------------------------------------------------------
B. PAGE / URL MAPPING (environment variables → Siebel view)
--------------------------------------------------------------------------------
  DMS_REAL_URL_CONTACT     → Contact / Find view (mobile search, initial + re-find)
  DMS_REAL_URL_ENQUIRY     → Enquiry view (optional; booking goto; skip_find entry)
  DMS_REAL_URL_VEHICLE     → Auto Vehicle List / stock search (key/chassis/engine)
  DMS_REAL_URL_VEHICLES    → In-transit / receipt view (Process Receipt)
  DMS_REAL_URL_PRECHECK    → Pre Check view (optional; may match PDI URL)
  DMS_REAL_URL_PDI         → PDI view (Pre Check + PDI submit per config)
  DMS_REAL_URL_LINE_ITEMS  → Order line / allotment (Price All, Allocate)
  DMS_REAL_URL_REPORTS     → Loaded into urls object; not used in linear SOP core

--------------------------------------------------------------------------------
C. LINEAR SOP — STAGES AND ACTIONS (default: skip_contact_find = false)
--------------------------------------------------------------------------------

-- Stage 1: Find customer by mobile --
  PAGE: DMS_REAL_URL_CONTACT (label: contact_find)
  ACTIONS:
    - Expand Find fly-in if collapsed (_try_expand_find_flyin)
    - Prepare Find applet for Contact object (_try_prepare_find_contact_applet)
    - Fill mobile ONLY in Find pane (CSS / semantic / DOM scan fallbacks)
    - Blur/settle
    - Click Find / Go query (_click_find_go_query)
    - Wait after Go (~2s initial find; same helper with 2s for re-find)

  DECISION — customer found?
    - If dms_contact_path == "new_enquiry" (case-insensitive): FORCED "not matched"
      → automation always treats as new enquiry path (Stage 2 runs).
    - Else: _siebel_ui_suggests_contact_match(page, mobile)
      → True  = "Customer found": mobile substring (last 10 digits preferred)
        appears in a <table><tbody><tr> with ≥3 <td> cells (not just the query field).
      → False = "Not found / no grid hit": proceed to basic enquiry (Stage 2).
      Note: div-only grids may false-negative (no table row match).

-- Stage 2: Create basic enquiry (only if NOT matched in Stage 1) --
  PAGE: Current Siebel view after Find (enquiry/customer applet on same session)
  SKIPPED if: Stage 1 matched (existing contact).
  FIELDS FILLED (_fill_basic_enquiry_details) — values from dms_values:
    - First Name, Last Name, Address (or Street), State (select), PIN/Postal/ZIP
  NOT filled here: father/husband, relation, landline (strict SOP).
  ACTION: Click Save toolbar (Save / Save Record / Commit) if found.

-- Stage 3: Mandatory re-find (only if Stage 2 ran — new basic enquiry saved) --
  PAGE: DMS_REAL_URL_CONTACT again (same as Stage 1, label: contact_find)
  ACTIONS: Same as Stage 1 mobile find (_refind_customer_after_enquiry).
  SKIPPED if: Stage 2 skipped (contact already existed).

-- Stage 4: Care-of (ALWAYS after Stages 1–3) --
  PAGE: Current view after find / re-find
  FIELDS (_fill_siebel_care_of_only):
    - Father / Husband / Parent inputs ← father_husband_name
    - Relation / S/O select ← relation_prefix
  ACTION: Save toolbar again.

-- Stage 5: Vehicle list search and scrape (stage_5_vehicle_flow) --
  PAGE: DMS_REAL_URL_VEHICLE (label: vehicle_list)
  FIELDS (at least one must succeed or flow errors):
    - Key #     ← key_partial
    - Chassis / Frame / VIN ← frame_partial
    - Engine    ← engine_partial
  ACTION: Find/Go on vehicle search; wait networkidle (best effort); scrape first
          wide grid row (scrape_siebel_vehicle_row).

  DECISION — In Transit?
    - Scraped row text scanned for "in transit" / "in-transit" (_grid_cells_suggest_in_transit)
    - in_transit_state = True  → Stage 5b (receipt + pre-check + PDI)
    - in_transit_state = False → skip receipt/PDI branch

-- Stage 5b (only if In Transit): Receipt, Pre Check, PDI --
  PAGE (receipt): DMS_REAL_URL_VEHICLES if set (label: vehicles_receipt)
  ACTION: Click toolbar matching Process Receipt / Receive / GRN (best effort)

  PRE-CHECK / PDI (_siebel_run_precheck_and_pdi):
    - If PRECHECK and PDI URLs equal: one goto (pdi_precheck_same_url) → Pre Check
      complete click → PDI submit click
    - If only PRECHECK: goto precheck → Pre Check complete
    - If PDI set: goto pdi → (if no separate precheck URL, try Pre Check on PDI
      view first) → PDI submit
    Toolbar regexes include: "Complete Pre Check", "PDI Submit" (PDI-specific),
    etc.

-- Stage 6: Generate Booking (ALWAYS after Stage 5 / 5b, both In Transit and not) --
  PAGE: DMS_REAL_URL_ENQUIRY if set, else DMS_REAL_URL_CONTACT (label: enquiry_for_booking)
        If neither set: stays on current page (logged in notes).
  ACTION: Click toolbar "Generate Booking" / "Generate Book" (_try_click_generate_booking)

-- Stage 7: Allotment / line items (ONLY if NOT In Transit) --
  PAGE: DMS_REAL_URL_LINE_ITEMS (label: line_items_allotment)
  SKIPPED if: in_transit_state is True
  ACTIONS: Price All, then Allocate / Allocate All (best-effort toolbar clicks)

-- Stage 8: Invoice (hook only — no Playwright clicks) --
  Logs only: "Invoice step pending (not automated)." / step "Ready for invoice creation."

--------------------------------------------------------------------------------
D. ALTERNATE ENTRY: skip_contact_find (skip_find path)
--------------------------------------------------------------------------------
  Stage 1 (Contact Find) is BYPASSED.

  PAGE: DMS_REAL_URL_ENQUIRY or fallback DMS_REAL_URL_CONTACT (label: enquiry_or_contact)
  ACTIONS:
    - Fill mobile on enquiry form (_try_fill_mobile_on_enquiry_form)
    - _fill_basic_enquiry_details (first, last, addr, state, pin)
    - If landline present: Work / Alternate / Landline fields
    - Save
    - MANDATORY: _refind_customer_after_enquiry on DMS_REAL_URL_CONTACT
    - stage_4_add_care_of (same as main path)
  Then continues with Stage 5 onward (vehicle → booking → allotment → invoice hook).

--------------------------------------------------------------------------------
E. OUTPUT / SIDE EFFECTS (API / operator visibility)
--------------------------------------------------------------------------------
  Returns dict with: vehicle (scraped row), error, dms_siebel_forms_filled,
  dms_siebel_notes, dms_milestones, dms_step_messages.
  dms_siebel_forms_filled = True if any Save toolbar click succeeded on customer
  steps (Stages 2 and/or 4, or skip_find save).

--------------------------------------------------------------------------------
F. ERRORS / EARLY EXIT (high level)
--------------------------------------------------------------------------------
  - Missing DMS_REAL_URL_CONTACT (normal path Stage 1)
  - Mobile field not found on contact view
  - Mandatory re-find failed after new enquiry
  - skip_find: missing enquiry URL, mobile on form failed, missing contact URL for re-find
  - Missing DMS_REAL_URL_VEHICLE
  - Vehicle view: no key/chassis/engine inputs found
  - PlaywrightTimeout / RuntimeError (e.g. browser closed) / generic Exception
